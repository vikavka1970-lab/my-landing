## РАЗДЕЛ 2 — СТРУКТУРА ИДЕАЛЬНОГО ЛЕНДИНГА

Психологическая логика: **ВНИМАНИЕ → ИНТЕРЕС → ДОВЕРИЕ → ЖЕЛАНИЕ → ДЕЙСТВИЕ**

```
Hero → Доверие → Услуги → Портфолио → Кейс → О себе → Процесс → Отзывы → FAQ → CTA
```

---

### Блок 1 — HERO (Первый экран)
**Цель:** Захватить внимание за 3 секунды

**Контент:**
- H1: чёткая ценность — кто ты + что делаешь + для кого
- Подзаголовок: 1-2 предложения о результате для клиента
- Два CTA: «Посмотреть проекты» (браузеры) + «Обсудить задачу» (готовые покупатели)
- Визуал: анимация / 3D-сцена / фото + availability badge («Доступен для проектов — Q2 2026»)
- Навигация: sticky, минималистичная

---

### Блок 2 — SOCIAL PROOF BAR (Полоса доверия)
**Цель:** Мгновенный кредит доверия до того, как посетитель что-то прочитал

**Контент:**
- Логотипы 5-8 клиентов (grayscale → color при hover)
- ИЛИ: строка статистики — «35+ проектов · 5+ лет · 3 континента»
- ИЛИ: одна сильная цитата клиента с фото

**Конверсионная роль:** Посетители, видящие социальные доказательства в начале, прокручивают дальше в 2-3× чаще

---

### Блок 3 — УСЛУГИ (три направления)
**Цель:** Помочь посетителю понять, что именно ему нужно

**Контент:**
- **Сайты:** Типы, стек, «для бизнесов, которым нужна digital-витрина или SaaS MVP»
- **Приложения:** iOS/Android/кросс-платформа, «от идеи до App Store»
- **ИИ-агенты:** Автоматизация, чат-боты, пайплайны, «ваш бизнес работает 24/7 без людей»
- 2-3 буллета + иконки, не более 30 слов на услугу

---

### Блок 4 — ПОРТФОЛИО
**Цель:** Визуальное доказательство компетентности

**Контент:**
- 4-6 проектов (микс: сайт + приложение + ИИ)
- Каждая карточка: скриншот/мокап, 1 строка описания, 2-3 метрики результата, стек-теги
- Фильтры: Все | Сайты | Приложения | ИИ-агенты
- Bento-grid или равномерная сетка

---

### Блок 5 — ДЕТАЛЬНЫЙ КЕЙС
**Цель:** Показать стратегическое мышление, ROI-результаты

**Контент:**
- Проблема → Решение → Результат (до/после с цифрами)
- Технологии, цитата клиента, конкретные метрики

---

### Блок 6 — О СЕБЕ
**Цель:** Построить личное доверие — люди нанимают людей

**Структура (150-250 слов):**
1. Крючок: кто ты + почему это важно для клиента
2. Мост доверия: опыт, тип клиентов, конкретный результат, что отличает
3. Человеческий элемент: стиль работы, ценности
4. Переход к CTA

---

### Блок 7 — ПРОЦЕСС РАБОТЫ
**Цель:** Снять тревогу о сотрудничестве

**Контент:**
1. Бриф / Discovery Call — «Без обязательств»
2. Предложение и объём — «Чёткий план и цифры»
3. Разработка итерациями — «Живые превью и чекины»
4. Запуск и передача — «Документация и пост-запуск поддержка»

---

### Блок 8 — ОТЗЫВЫ
**Цель:** Независимая валидация в момент принятия решения

**Контент:**
- 3-5 отзывов: имя + должность + компания + фото + **специфичная цитата с результатом**
- Ссылка на LinkedIn клиента (подтверждает подлинность)
- Можно: автоматически прокручивающаяся «Стена любви» в стиле Twitter-карточек

---

### Блок 9 — FAQ
**Цель:** Снять последние барьеры перед принятием решения

**Темы для 6-8 вопросов:**
- Сроки и процесс
- Ценообразование / как это работает
- Поддержка после запуска
- Как работают ИИ-агенты (для нетехнической аудитории)
- Чем отличаешься от агентства
- Возможность работы с существующей командой

---

### Блок 10 — CTA / КОНТАКТ
**Цель:** Конвертировать намерение в действие

**Контент:**
- Заголовок: «Готов начать? Давай поговорим»
- Форма: имя + email + тип проекта + описание (3-5 полей максимум)
- Calendly-ссылка — прямое бронирование без переписки
- Email + мессенджеры
- «Отвечу в течение 24 часов» — обещание ответа
- Текущий статус доступности

## РАЗДЕЛ 3 — ТРЕНДЫ 2025–2026

### 10 применимых трендов

---

#### 1. BENTO GRID LAYOUT
**Вид:** Асимметричная сетка карточек разного размера — как Apple, Linear, Notion
**Применение:** Секция портфолио или навыков — разноразмерные карточки, hover-анимации
**Примеры:** Apple, Vercel, Linear.app
**Сложность:** Легко (CSS Grid)

---

#### 2. DARK THEME С НEOН-АКЦЕНТАМИ
**Вид:** Тёмные поверхности (не чисто чёрные, с глубиной) + electric violet/cyan акценты
**Применение:** Базовая тема + неоновый CTA + toggle для светлой
**Примеры:** Vercel, Railway.app, Resend.com, Raycast.com
**Сложность:** Легко

---

#### 3. GLASSMORPHISM 2.0 (Сдержанный)
**Вид:** Матовое стекло: `backdrop-filter: blur()`, 1px прозрачная рамка, тонкое свечение
**Применение:** Навигация, контактная форма, hero tagline-панель (1-2 элемента, не весь сайт)
**Примеры:** macOS Sonoma, Linear command palette
**Сложность:** Легко

---

#### 4. AURORA / MESH GRADIENT BACKGROUNDS
**Вид:** Мягкие цветные блобы (purple, teal, pink) медленно дрейфуют на тёмном фоне
**Применение:** Hero-фон — CSS радиальные градиенты + `filter: blur(80px)` + анимированное движение
**Примеры:** Stripe, Runway ML, Resend.com
**Сложность:** Легко (CSS) / Высокая (WebGL)

---

#### 5. КИНЕТИЧЕСКАЯ ТИПОГРАФИКА (Text as Hero)
**Вид:** Гигантские буквы анимируются по скроллу или на загрузке; шрифт — главный визуал
**Применение:** Hero: «Я строю [сайты / приложения / ИИ-агентов]» — каждое слово появляется последовательно
**Примеры:** Iuri de Paula, Bruno Simon, Lusion.co
**Сложность:** Средняя

---

#### 6. SCROLL-TRIGGERED МИКРОАНИМАЦИИ
**Вид:** Элементы не просто «фейдятся» — вращаются на 5°, масштабируются 95%→100%, открываются clip-path
**Применение:** Карточки проектов «влетают» с разных сторон; заголовки секций открываются wipe-эффектом
**Примеры:** Stripe.com, Superhuman.com
**Сложность:** Средняя (GSAP ScrollTrigger / Framer Motion)

---

#### 7. AI-NATIVE «ЖИВОЙ» ЭЛЕМЕНТ
**Вид:** Встроенный чат-виджет или живое демо ИИ-агента прямо на странице
**Применение:** «Задай вопрос моему ИИ-ассистенту» — предпромпт с bio, услугами, кейсами
**Примеры:** Emerging 2025 pattern — ai.pika.art, Luma AI
**Сложность:** Высокая (Claude/GPT API + rate limiting)
**Почему работает:** Показываешь, а не рассказываешь — самый сильный trust-сигнал для AI-специалиста

---

#### 8. ПЕРЕМЕННЫЕ ШРИФТЫ (Variable Fonts)
**Вид:** Вес/ширина шрифта меняются при скролле или hover; один файл = все начертания
**Применение:** Заголовки «утолщаются» при скролле, адаптивная типографика
**Сложность:** Средняя

---

#### 9. SPLINE 3D EMBEDS
**Вид:** Интерактивный 3D-объект (глобус, абстрактная форма, логотип) реагирует на мышь
**Применение:** Hero — Spline `<spline-viewer>` без WebGL-экспертизы
**Примеры:** Spline.design, Jitter.video, Basement Studio
**Сложность:** Средняя (Spline no-code) / Высокая (Three.js custom)

---

#### 10. SOCIAL PROOF MOSAIC («Стена любви»)
**Вид:** Карточки в стиле Twitter бегут двумя рядами в разные стороны (CSS marquee)
**Применение:** Секция отзывов — автопрокрутка, пауза при hover, logo-ticker под ней
**Примеры:** Resend.com, Cal.com, Shadcn UI
**Сложность:** Легко

### Рекомендуемый стек

```
Layer 1 — CSS:           hover, transitions, grain, aurora bg
Layer 2 — Framer Motion: page transitions, whileInView, AnimatePresence
Layer 3 — GSAP:          ScrollTrigger sequences, SplitText, magnetic buttons
Layer 4 — Lenis:         smooth scroll foundation (интегрировать с GSAP)
Layer 5 — Optional:      Three.js / Spline (только десктоп, с mobile fallback)
```

**Фреймворк:** Next.js + React

**НЕ смешивать:** GSAP и Framer Motion на одних DOM-элементах — конфликт!

---

### Сравнение библиотек

| Библиотека | Лучше для | Сложность | Размер |
|---|---|---|---|

| **Three.js / R3F** | 3D, WebGL, частицы | Высокая | ~130KB |
| **Spline** | 3D без WebGL кода | Легко | iframe |
| **tsParticles** | Particle backgrounds | Легко | ~20KB |
| **CSS only** | Базовые эффекты | Легко | 0KB |

## РАЗДЕЛ 7 — ЭКСПЕРТНАЯ ОЦЕНКА

> Три независимые точки зрения на проект: лендинг-портфолио цифрового мастера
> (сайты · приложения · ИИ-агенты)

---

### 🎨 ЭКСПЕРТ 1 — UX-дизайнер (10 лет опыта)

---

#### Типичные ошибки новичков в структуре

**1. «Я-центричный» лендинг вместо «клиент-центричного»**
Самая частая ошибка — страница рассказывает о тебе: «Я умею», «Я делаю», «Мои навыки». Клиент приходит с вопросом «Что это даст МНЕ?». Каждый блок должен отвечать на этот вопрос.

**2. Перегруженный герой**
Новички пихают в first screen: логотип, навигацию, фото, три заголовка, список навыков, кнопку и счётчики. Итог — посетитель не знает, куда смотреть. Правило: в герое должно быть ровно одно сообщение и один главный CTA.

**3. Портфолио без контекста**
Скриншоты без объяснений — не кейсы. «Вот картинка сайта» не убеждает. Нужно: задача клиента → твоё решение → измеримый результат. Без этого кейс — просто галерея.

**4. Отсутствие визуальной иерархии**
Одинаковый размер шрифтов, одинаковый вес, одинаковые отступы — глаз не знает, что читать первым. H1 должен быть очевидно главным, H2 второстепенным, а body — фоном.

**5. «Кладбище навыков»**
Список из 40 технологий с иконками никого не впечатляет — только пугает. Клиент не знает половину из них и чувствует себя некомпетентным. Лучше: 3 категории с 3-4 ключевыми технологиями и объяснением зачем они нужны клиенту.

**6. Нет мобильной версии в приоритете**
Более 60% трафика — мобильный. Дизайн «сначала декстоп» даёт корявую мобилку. Проверяй каждый блок на телефоне до публикации.

**7. Форма на 10 полей**
Длинная форма контакта убивает конверсию. Каждое лишнее поле снижает заполняемость на ~10-15%. Максимум 4-5 полей: имя, email, тип проекта, описание.

**8. Нет sticky CTA**
Посетитель читает страницу и хочет написать — но кнопка только в футере. Зафиксированная навигация с кнопкой «Обсудить проект» должна быть видна всегда.

---

#### Что ОБЯЗАТЕЛЬНО должно быть

| Элемент | Почему критично |
|---|---|
| **Чёткий заголовок H1** в герое | Если за 3 сек непонятно что ты делаешь — уходят |
| **Реальные кейсы с результатами** | Скриншоты без контекста не убеждают |
| **Фото или видео** (лицо специалиста) | Доверие к человеку выше чем к аватару |
| **Социальные доказательства** именные | Анонимные отзывы не работают |
| **Процесс работы** (как выглядит сотрудничество) | Снимает главный страх: «а вдруг будет хаос» |
| **Чёткий CTA** на каждом экране | Посетитель не должен «искать как написать» |
| **Время отклика** («отвечу в 24ч») | Снимает тревогу об игнорировании |
| **Мобильная версия** полноценная | 60%+ трафика, нет прощения за кривой мобайл |

---

#### Что раздражает и заставляет уйти

**Моментальные убийцы конверсии:**
- Долгая загрузка (>3 секунд) — 40% уходят немедленно
- Автоматически запускающаяся музыка или видео со звуком
- Агрессивный поп-ап на 2-й секунде: «Подпишитесь!»
- Горизонтальный скролл на мобиле — ощущение поломки

**Ошибки доверия:**
- Стоковые фото людей вместо реального фото специалиста
- «Copyright 2019» в футере — выглядит как заброшенный сайт
- Нет ни одного реального имени в отзывах
- Ссылки ведут на несуществующие страницы (404)

**Когнитивная перегрузка:**
- Слишком много анимаций одновременно — мозг перегружается и уходит
- Блоки текста без воздуха — «простыни» по 10 строк без абзацев
- 5+ шрифтов или 6+ цветов — хаос, а не дизайн
- Непоследовательный стиль — каждый блок в «своём» дизайне

**Потеря доверия:**
- Заявленные навыки не совпадают с портфолио (говоришь «делаю приложения» — ни одного в кейсах)
- Общие слова без доказательств: «качественно», «профессионально», «быстро» — кто так не говорит?
- Цены не указаны И нет объяснения почему (воспринимается как «скрывают, значит дорого»)

---

### 📊 ЭКСПЕРТ 2 — Маркетолог, специалист по конверсии

---

#### Оффер, который зацепит клиента

**Проблема типичных портфолио-офферов:**
«Я — разработчик. Вот мои работы. Напишите мне.» — это не оффер, это резюме.

**Формула сильного оффера для цифрового мастера:**

```
[Конкретный результат] для [конкретной аудитории]
за [конкретные условия / без [главной боли]
```

**Примеры:**
> «Сайт под ключ за 3 недели — от брифа до первого клиента»
> «Автоматизирую ваш отдел продаж ИИ-агентом за 2 недели. Без технических знаний с вашей стороны»
> «Строю MVP для стартапов — от идеи до App Store за 6 недель»

**Почему три услуги — это сила, если правильно подать:**
Обычно три услуги читаются как «я не определился». Но можно перевернуть: **«Я — единственная точка сборки вашего цифрового продукта»**. Один человек видит всю экосистему: сайт + приложение + автоматизация работают как единый организм, а не три несвязанных части.

---

#### Триггеры доверия (по убыванию силы)

| # | Триггер | Как реализовать |
|---|---|---|
| 1 | **Конкретные результаты с цифрами** | «Конверсия выросла с 1.2% до 3.8%», «Сэкономил 40ч/неделю» |
| 2 | **Узнаваемые клиенты / бренды** | Логотипы даже одного известного клиента меняют всё |
| 3 | **Именные отзывы с LinkedIn-ссылкой** | Верифицируемость = максимум доверия |
| 4 | **Лицо специалиста + история** | Люди покупают у людей, а не у сайтов |
| 5 | **Конкретный процесс работы** | Предсказуемость снимает риск в восприятии клиента |
| 6 | **Число завершённых проектов** | «35 проектов» > «много лет опыта» |
| 7 | **Гарантии или обещания** | «Правки до утверждения», «Отвечу в 24ч» |
| 8 | **Публичная активность** | GitHub, статьи, выступления — доказывают экспертизу |

---

#### Путь посетителя: от «зашёл» до «оставил заявку»

```
МОМЕНТ 0 — Первые 3 секунды (Hero)
  ↓
  Вопрос посетителя: «Это то, что я ищу?»
  Ответ страницы: Чёткий H1 + визуал + один CTA
  Если ДА → прокручивает дальше
  Если НЕТ → уходит (нельзя исправить)

МОМЕНТ 1 — Первые 15 секунд (Hero + Social Proof Bar)
  ↓
  Вопрос: «Можно ли ему доверять?»
  Ответ: Логотипы клиентов / цифры / один сильный отзыв
  Микроконверсия: решение остаться и читать

МОМЕНТ 2 — 30-90 секунд (Услуги + Портфолио)
  ↓
  Вопрос: «Делает ли он то, что нужно мне?»
  Ответ: Карточки услуг (самоидентификация) + кейсы с результатами
  Микроконверсия: «Похоже на мою задачу»

МОМЕНТ 3 — 2-4 минуты (Кейс + О себе + Процесс)
  ↓
  Вопрос: «Как это будет работать? Что если пойдёт не так?»
  Ответ: История проекта до/после + процесс работы по шагам
  Микроконверсия: снятие страха «буду работать вслепую»

МОМЕНТ 4 — 4-6 минут (Отзывы + FAQ)
  ↓
  Вопрос: «Другие уже пробовали? Что с ценой/сроками?»
  Ответ: Именные отзывы с результатами + FAQ по возражениям
  Микроконверсия: снятие финального возражения

МОМЕНТ 5 — CTA
  ↓
  Посетитель готов. Нужен минимальный friction.
  Форма 4 поля + Calendly-ссылка + обещание ответа
  КОНВЕРСИЯ: оставил заявку / забронировал звонок
```

**Ключевые точки потери на пути:**
- Hero не отвечает на вопрос → уход 60% до скролла
- Нет кейсов с результатами → уход на этапе «может быть»
- FAQ не закрывает вопрос цены → уход на финальном шаге
- Форма длинная / не работает → уход в 1 клике от конверсии

---

#### Тактики повышения конверсии

**Срочность и дефицит (без манипуляции):**
- «Беру 2-3 проекта в месяц — сейчас есть одно место» (если правда)
- «Сейчас доступен для старта в мае» — конкретность = доверие

**Снижение барьера первого контакта:**
- Calendly с конкретными слотами — убирает «буду переписываться»
- «Бесплатный 30-минутный звонок» — нет финансового риска
- «Отвечу в течение 24 часов» — убирает страх быть проигнорированным

**Сегментация CTA по намерению:**
- Холодный: «Посмотреть проекты» → в портфолио
- Тёплый: «Узнать о процессе» → в блок процесса
- Горячий: «Обсудить проект» → сразу в форму

### 👤 ЭКСПЕРТ 3 — Владелец бизнеса (потенциальный клиент)

> *Внутренний монолог типичного клиента, который заходит на портфолио-лендинг*

---

#### Первые 5 секунд: на что смотрит клиент

**Глаз сначала ловит (в порядке):**
1. Заголовок — «Кто это и что он делает?»
2. Визуал — «Выглядит ли это профессионально?»
3. Знакомые логотипы / цифры — «Кто ещё ему доверял?»
4. CTA-кнопку — «Что меня просят сделать?»

**Внутренние вопросы за 5 секунд:**
- «Делает ли он то, что мне нужно?» (релевантность)
- «Выглядит ли это как серьёзный человек?» (доверие)
- «Не трачу ли я время?» (ценность)

**Если на эти три вопроса нет быстрого ответа — закрываю вкладку.**

---

#### Что заставит ОСТАТЬСЯ

**«О, это именно то, что мне нужно»**
Когда в описании услуги узнаю свою проблему дословно. Не «разработка ИИ-агентов», а «автоматизирую ответы на однотипные запросы клиентов — ваша команда перестаёт тратить на это 3 часа в день».

**«Он уже делал похожее»**
Кейс с задачей, похожей на мою. Не важно — та же отрасль или нет. Важно: та же проблема, понятное решение, конкретный результат.

**«Реальный человек, не робот»**
Фото. Живой текст «от себя». Немного личности — не официоз. Я покупаю не технологию, я покупаю человека, которому доверю деньги и время.

**«Понятно как это работает»**
Блок «Как мы будем работать» с 4-5 шагами. Я хочу знать: что будет после того как напишу. Неизвестность — главный тормоз для решения.

**«Другие не пожалели»**
Не просто «спасибо за работу» — а конкретно: «До этого мы тратили 20 часов в неделю на обработку заявок. Теперь ИИ делает это за нас». И имя, должность, ссылка на компанию.

---

#### Что заставит ЗАКРЫТЬ вкладку

**Технические сигналы тревоги:**
- Страница грузится дольше 3 секунд — у меня мало времени, пошёл к следующему
- На мобиле текст вылезает за край — если своя страница кривая, что будет с моей?
- Ссылки не работают — небрежность в деталях = небрежность в проекте

**Доверительные сигналы тревоги:**
- «Последний проект — 2022 год» — не обновляет портфолио, значит нет новых клиентов
- Отзывы без имён и фотографий — выглядит как придуманные
- Пишет «10 лет опыта», в портфолио 3 студенческих проекта — несоответствие
- Заголовок «Привет, меня зовут Иван и я люблю программирование с детства» — это твоя биография, а не моя проблема

**Психологические триггеры ухода:**
- Слишком много всего одновременно — не могу сфокусироваться, устал, ухожу
- Непонятно сколько это стоит и как вообще начать — лень разбираться
- «Работаю с корпорациями и крупным бизнесом» — я небольшая компания, это не для меня
- Нет ответа на вопрос «сколько времени займёт?» — значит не знает или скрывает

**Главный момент истины для бизнес-клиента:**
> Мне нужно понять за 2 минуты: **решит ли этот человек мою конкретную проблему, в мои сроки, за разумные деньги, и не создаст ли новых проблем в процессе.** Если страница не отвечает на это — я не буду писать, чтобы «уточнить».

### 8.2 — ОБЯЗАТЕЛЬНЫЕ ЭКРАНЫ И UX-ПАТТЕРНЫ

#### Стандартная архитектура flow (5 экранов)

```
Экран 1 — Приветствие
  Логотип / фото · Краткое описание · Кнопка «Начать»

Экран 2 — Тип проекта
  Сайт / Приложение / AI-агент / Другое (radio cards)

Экран 3 — Детали (условный, зависит от экрана 2)
  Сроки · Бюджет · Главная задача

Экран 4 — Контакт
  Имя (из Telegram API) · Telegram / Email

Экран 5 — Подтверждение + Success
  Summary брифа · Кнопка «Отправить» · Анимация успеха
```

**Оптимальное число шагов: 3–5.** Больше 7 — потеря 40–50% пользователей.

---

#### Обязательные UI-элементы

| Элемент | Описание |
|---|---|
| **BackButton** | Нативная кнопка Telegram — показывать на всех экранах кроме первого |
| **MainButton** | Основная кнопка внизу (полная ширина) — «Далее» / «Отправить» |
| **Progress bar** | Линия или «2/5» вверху — обязательно, без него пользователь не знает конца |
| **Smart defaults** | Имя из `tg.initDataUnsafe.user` — не просить то, что Telegram уже знает |
| **Inline validation** | Ошибки под полем в реальном времени, не после Submit |
| **Loading state** | Spinner + disabled кнопка во время отправки |
| **Success screen** | Анимация + текст + автозакрытие через 2–3 сек |

---

#### Паттерны повышения конверсии

- **One field per screen** — показывать 1–2 поля за раз, не всё сразу
- **Progressive disclosure** — показывать подкатегории только после выбора типа
- **Estimated time** — «Займёт около 2 минут» в начале
- **Правильные input type** — `tel`, `email`, `number` — правильная клавиатура на телефоне
- **Минимализм в тексте** — «Ваш бюджет» вместо «Пожалуйста, укажите ваш бюджет»
- **Динамический CTA** — «Ещё 2 вопроса» вместо просто «Далее»

---

#### Ключевые ограничения Telegram Mini App

| Ограничение | Workaround |
|---|---|
| Нет доступа к файловой системе | Загрузка через `input type="file"` или ссылка на облако |
| Date picker работает нестабильно | Использовать custom picker |
| Только одна MainButton | Менять текст под контекст шага |
| Нет Service Workers | Данные хранить на сервере, не в localStorage |
| iOS/Android разное поведение | Тестировать на обоих |
| `:hover` не работает на touch | Использовать `:active` для мобильных стилей |

### 8.3 — ТЕХНИЧЕСКИЙ СТЕК И АРХИТЕКТУРА

#### Рекомендуемый стек

```
Фронт:    HTML / CSS / JS (нативный) + Telegram WebApp SDK
Бэкенд:   Node.js + Telegraf
Деплой:   Vercel (фронт + serverless функция вебхука)
Бесплатно: Vercel Free + Telegram API + GitHub Pages
```

#### Как работает передача данных

```
Пользователь заполняет форму в Mini App
          ↓
tg.sendData(JSON.stringify(briefData))
          ↓
Telegram доставляет событие web_app_data в бот
          ↓
Telegraf: bot.on('web_app_data', ctx => { ... })
          ↓
Бот отправляет подтверждение пользователю + уведомление администратору
```

#### Сравнение библиотек для бота

| Библиотека | Для кого | Вердикт |
|---|---|---|
| **node-telegram-bot-api** | Новички, быстрый старт | Просто, но мало возможностей |
| **Telegraf** | Баланс простоты и функций | ✅ Рекомендуется |
| **Grammy** | TypeScript-проекты | Лучшая документация, молодой |

**Выбор для этого проекта: Telegraf** — проверенная библиотека, хорошо работает на Vercel, много примеров.

#### Ключевые методы Telegram WebApp SDK

```javascript
const tg = window.Telegram.WebApp;

tg.expand()                          // Развернуть на весь экран
tg.initDataUnsafe.user               // Данные пользователя (имя, id)
tg.sendData(JSON.stringify(data))    // Отправить бриф в бота
tg.BackButton.show()                 // Показать кнопку «Назад»
tg.MainButton.text = 'Далее'        // Текст основной кнопки
tg.close()                           // Закрыть Mini App
tg.HapticFeedback.notificationOccurred('success') // Вибрация
```
