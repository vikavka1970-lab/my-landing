### Два режима тестирования

**Режим A — браузер** (без Telegram SDK)  
Открыть https://vp-brief-bot.vercel.app в браузере.  
- `hasTg = false` — имя пользователя показывается как «Гость», ник не заполняется  
- `tg.sendData()` не вызывается (ошибка поймана в try/catch)  
- Вместо нативных кнопок Telegram видны HTML fallback-кнопки (`.fb-btn`)  
- Навигация «Назад» — кнопка «‹ Назад» внутри экрана

**Режим B — Telegram** (полноценный)  
Открыть бота @Lendingferst_bot → нажать кнопку меню внизу.  
- `hasTg = true` — имя и ник берутся из профиля Telegram  
- Нижняя MainButton — нативная кнопка Telegram  
- Стрелка BackButton — в хедере Telegram  
- Кнопка «‹ Назад» внутри экрана тоже видна (оба элемента навигации активны)  
- `tg.sendData()` вызывается при отправке, но данные пока некуда получать (Node.js-бот не написан)

### 2.11 Тема оформления

Mini App использует CSS-переменные Telegram SDK:
```
--tg-theme-bg-color            → фон
--tg-theme-secondary-bg-color  → карточки
--tg-theme-text-color          → текст
--tg-theme-hint-color          → подсказки
--tg-theme-button-color        → accent (кнопки, border выбранных карточек)
```

**Проверить:**
- [ ] В светлой теме Telegram — приложение светлое
- [ ] В тёмной теме Telegram — приложение тёмное
- [ ] В браузере (без SDK): используются fallback-значения (`#ffffff`, `#f4f4f5`, `#000000`)

## Известные ограничения (не баги)

| Ситуация | Поведение | Причина |
|---|---|---|
| `tg.sendData()` не доставляет данные | Нормально | Node.js-бот не написан; данные уходят в Telegram, но никто их не принимает |
| В браузере имя = «Гость», ник пустой | Нормально | Telegram SDK не доступен вне Telegram |
| В браузере кнопки HTML, не нативные | Нормально | Fallback для тестирования (`hasTg = false`) |
| В Telegram видны ДВЕ кнопки «Назад» | Известно | Нативная стрелка BackButton + кастомная `.back-btn` — оба рабочие |
