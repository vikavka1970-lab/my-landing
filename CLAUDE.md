## Архитектура

- **Несколько HTML-файлов** — каждая страница живёт в отдельном `.html` (HTML + CSS + JS внутри)
  - `index.html` — главный лендинг
  - `pricing.html` — страница тарифов
- **Никаких внешних библиотек** — только нативный HTML/CSS/JS
- **Никаких фреймворков** — без React, Vue, Tailwind и т.п.
- Исключение: Google Fonts подключены через `<link>` в `<head>`
- Исключение: Web3Forms для формы обратной связи (один `fetch`)
- Dev-зависимость: `puppeteer` (`package.json`) — для автоматических скриншотов портфолио. В продакшн-сборку не попадает.

**При создании новой страницы** — обязательно копировать блок с CSS-переменными, шрифтами, grain texture, cursor, кнопками и fade-анимациями из существующей страницы. Все страницы должны выглядеть единым целым.

## Дизайн

**Тёмная тема — обязательно.** Светлую тему не вводить.

CSS-переменные (не менять без явной просьбы):
```css
--bg:      #0F172A   /* фон страницы */
--surface: #1E293B   /* карточки, блоки */
--border:  #334155   /* границы */
--accent:  #6366F1   /* основной акцент — indigo/синий */
--cyan:    #06B6D4   /* вторичный акцент — cyan */
--text:    #F1F5F9   /* основной текст */
--muted:   #94A3B8   /* второстепенный текст */
```

**Шрифты:**
- `Syne` — заголовки, кнопки, навигация
- `DM Sans` — основной текст
- `Syne Mono` — метки, теги

## Адаптивность

- Сайт должен хорошо выглядеть на телефоне, планшете и десктопе
- Основные breakpoints: `640px`, `860px`, `1024px`
- При любых новых секциях или компонентах сразу добавлять `@media`
- Проверять: нет ли горизонтального скролла на мобайле

---

## Стиль текста на сайте

- Простой, живой, дружеский — как разговор с умным коллегой
- Без канцелярита: не «осуществляю разработку», а «делаю сайты»
- От первого лица: «я», «со мной», «напишите мне»
- Акцент на результате для клиента, не на процессе
- Язык: русский

## Целевое действие

Сайт должен приводить к одному из двух действий:
1. **Написать в Telegram** — `@viktoriapoltavchenko`
2. **Заполнить форму** — секция `#contacts` на `index.html` (отдельной страницы контактов пока нет)

Любая новая секция, кнопка или страница оценивается с точки зрения: *помогает ли это посетителю сделать шаг к контакту?*

---

## Архитектура CSS

- Стили пишутся внутри каждого `.html`-файла — общего `.css`-файла нет
- Имена классов — в kebab-case: `.pricing-card`, `.nav-inner`, `.card-title`
- Структура блока стилей внутри файла: сначала reset/base → типографика → layout → компоненты → секции → адаптивность
- Если одинаковый стиль нужен на двух страницах — копировать, не выносить в общий файл (пока страниц меньше пяти)

---

## Навигация между страницами

- Логотип «ВП» всегда ведёт на `index.html`
- Ссылки в nav на внутренних страницах (не `index.html`) ссылаются как `index.html#section`
- Кнопка «Обсудить проект» всегда ведёт на `index.html#contacts`

## Деплой

Сайт хостится на GitHub Pages. Всё что есть в ветке `master` — автоматически публикуется.

```
git add <файлы>
git commit -m "описание изменений"
git push origin master
```

Изменения появляются на сайте через 1–2 минуты после пуша.

## Правила работы

**Перед созданием нового файла** — сначала объяснить что планирую сделать и зачем, дождаться подтверждения.

**Перед крупными изменениями в `index.html`** — описать план, не молча переписывать большие блоки.

**После каждого значимого изменения** — делать `git commit` и `git push origin master`.

**Никогда не добавлять:**
- Фейковые отзывы или выдуманные цифры клиентов
- Счётчики проектов без реальных данных
- Что-либо, что создаёт ложное впечатление об опыте
